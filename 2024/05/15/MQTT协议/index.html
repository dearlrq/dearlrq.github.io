—–Python—–

1- 三种身份

  • 发布者（Publisher）

  • 服务器（Broker）

  • 订阅者（Subscriber）

  • 他们通过主题进行链接，只要主题相同，就能传输信息。

2- 消息结构

  • 主题（Topic）：字符串

  • 负载（Payload）：字符串

3- 搭建mqtt-broker (中心服务器)

3-1 准备

  • 登陆一台作为服务器的 Ubuntu 主机 (本地主机或云服务器) 。

3-2 下载 EMQX 安装包 (命令)

  • wget https://www.emqx.com/zh/downloads/broker/5.0.11/emqx-5.0.11-ubuntu18.04-amd64.deb

3-3 安装 (命令)

  • sudo apt install ./emqx-5.0.11-ubuntu18.04-amd64.deb

3-4 运行 (命令)

  • sudo systemctl start emqx

3-5 非必选 (命令)

  • sudo emqx ctl admins add [ 用户名 ] [ 用户密码 ]

3-6 控制台

  • http://[ broker 的 ip ]:18083/

4- mqtt接口的python实现 (消息的发送端和接收端)

4-1 安装python依赖库

  • pip install paho-mqtt

4-2 连接到broker

  • 1、from paho.mqtt import client as paho_mqtt_cient

  • 2、建立Client对象

    • _client = paho_mqtt _client.Client (client _id)

    • client_id是它的名字

  • 3、链接服务器IP和端口号

    • _client.connect (broker-ip, broker-port)
    • broker-ip：服务器IP
    • broker-port：服务器端口号，一般是1883

  • 4、判断是否连接成功

    • 先写一个回调函数

      • def on_connect (client, userdata, flags, rc):
      • if rc == 0:
      • 连接上了
      • else:
      • 没连接上

    • 然后绑定到Client

      • _client.on_connect = on_connect

  • 5、开启线程

    • self._client.loop_start ( )

4-3 消息发送端

  • result = client.publish (topic, msg, qos, retain)
    • topic：消息名、主题
    • msg：消息内容、负载
    • retain：状态机消息
    • 返回消息状态

4-4 消息接收端

  • 1、写一个回调函数
    • def default_on_message (self, client, userdata, msg):
    • ……
    • 默认回调函数
    • print (msg.payload.decode (‘utf-8’))
  • 2、把回调函数绑定到_client.on_message
    • self.client.on_message = self.default_on_message
  • 3、关联到发送端消息名
    • self.client.subscribe (topic)
    • 绑定关联完成后，接收到同名消息，就会激活回调函数
  • 4、在回调函数中处理接收消息

4-5 关闭链接

  • _client.loop_stop ( ）
  • _client.disconnect ( )

—–Java—–

1- 准备工作

  • windows10-EMQX与MQTTX的安装及配置使用教程-CSDN博客

2- EMQX常用命令

  • .\emqx.cmd start ：启动服务和 Erlang 节点

  • .\emqx.cmd stop ：停止服务和 Erlang 节点

  • .\emqx.cmd restart ：运行停止命令和启动命令

  • .\emqx.cmd ping ： 检查节点是否正在运行

  • .\emqx.cmd ctl ：运行管理命令

  • .\emqx.cmd console ：在Windows shell 中启动 Erlang 版本

  • .\emqx.cmd attach ：连接到正在运行的节点并打开交互式控制台

  • .\emqx.cmd remote_console ：与附加相同

  • .\emqx.cmd list ： 显示已安装的 Erlang 服务的列表

  • .\emqx.cmd usage ：显示可用命令

3- 实际应用

3-1 应用场景

硬件采集的数据传入EMQX平台(采用MQTT协议)，java通过代码连接MQTT服务器，进行采集数据接收、解析、业务处理、存储入库、数据展示。

MQTT 是基于 发布(Publish)/订阅(Subscribe) 模式来进行通信及数据交换的。
